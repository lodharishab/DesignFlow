
# DesignFlow - Project Overview (Version 0.02)

This document provides a summary of the DesignFlow project, introducing authentication and basic user dashboards.

## I. Core Structure & Public Pages

The public-facing site is established.

*   **Homepage (`/`)**: Main landing page.
*   **Services Pages (`/design-services`, `/services/[serviceId]`)**: Service discovery pages.
*   **Portfolio Pages (`/portfolio`, `/portfolio/[id]`)**: Project showcase.
*   **Blog Pages (`/blog`, `/blog/[slug]`)**: Content marketing feature.
*   **Layout Components**: Navbar and Footer.

## II. Authentication & User Roles

This version introduces user authentication and role separation.

*   **Login (`/login`)**: A single login page with simulated role-based redirection.
*   **Client Signup (`/signup`)**: A form for clients to create an account.
*   **Designer Signup (`/signup/designer`)**: A separate form for designers to apply.
*   **Designer Pending Approval Page (`/designer/pending-approval`)**: A waiting page for new designers.

## III. Authenticated Areas (Placeholders)

Layouts and basic dashboard pages are created for authenticated users.

*   **Client Area (`/client/...`)**: A dashboard for clients with a sidebar layout.
*   **Designer Area (`/designer/...`)**: A dashboard for designers with a sidebar layout.
*   **Admin Area (`/admin/...`)**: A dashboard for administrators with a sidebar layout.

## IV. Technical Aspects

*   **Tech Stack**: Next.js, React, TypeScript, ShadCN UI, Tailwind CSS.
*   **Data Handling**: All content is powered by mock data.
